Introduction

In this little last extra section of the Guix Guide article series, we will briefly go over Guix Channels, how to set them up, and how to use them. Below is an example channels.scm file, this usually resides in ~/.config/guix/channels.scm:

Channels File


(append (list
         (channel
          (name 'jonabron)
          (branch "master")
          (url "https://github.com/librepup/jonabron.git"))
         (channel
          (name 'nonguix)
          (url "https://gitlab.com/nonguix/nonguix")
          (commit "48a8706d44040cc7014f36873dbd834c048aadd3")
          (introduction
           (make-channel-introduction
            "897c1a470da759236cc11798f4e0a5f7d4d59fbc"
            (openpgp-fingerprint

         (channel
           (name 'guix)
           (url "https://git.guix.gnu.org/guix.git")
           (commit "4baa120b8b298bec155c5b12c8068dda3c07df40")

This basic channels.scm file tells Guix where to look for channels. Channels are (usually) repositories which contain package definitions/recipes, system modules, services, and more! They can be incredibly useful, for example: the emacs channel includes freshly built melpa packages for GNU Emacs. This is useful for when an Emacs Package is not yet in the official guix repositories/guix channel.

I also have my own channel, the jonabron channel. This channel provides a discord package, the vicinae application launcher, the game osu!(lazer), and some other useful stuff you may want to install or use. Adding channels to your channels.scm file is relatively easy, the basic skeleton of a new channel looks like this:

Channel Skeleton


(channel
  (name 'guix) ; Put your Channels Name here.
  (url "https://git.guix.gnu.org/guix.git") ; The Repository URL goes here!
  (commit "4baa120b8b298bec155c5b12c8068dda3c07df40") ; Totally optional, but here you can specify a commit.
  (branch "master") ; Declare which branch of the repository to use.
  ; The channel introduction below is not needed for *every* channel,
  ; but some channels require or at least advocate for you specifying one.
  (introduction
    (make-channel-introduction
      "9edb3f66fd807b096b48283debdcddccfea34bad"
      (openpgp-fingerprint
       "BBB0 2DDF 2CEA F6A8 0D1D  E643 A2A0 6DF2 A33A 54FA"))))

And that's pretty much it! After specifying all the channels you want in your ~/.config/guix/channels.scm, simply run the command guix pull, and guix will update all the channels and rebuild itself. Note that a guix pull does not rebuild your entire system - it merely rebuilds the channel definitions that guix uses for commands such as guix install or guix search.

At last, you should run hash guix once the previous guix pull is finished. And then you're ready to go and use your newly added channels!
